# Contributing to YouTube Downloader Pro

First off, thank you for considering contributing to YouTube Downloader Pro! It's people like you that make this tool useful for everyone.

## 📝 Code of Conduct

Please be respectful and constructive in all interactions. We aim to create a welcoming environment for all contributors.

## 🚀 How to Contribute

### Reporting Bugs
- **Search existing issues** first to avoid duplicates
- Use the **bug report template** if available
- Include:
  - Clear description of the bug
  - Steps to reproduce
  - Expected vs actual behavior
  - Screenshots if applicable
  - Your OS and Python version

### Suggesting Features
- Open an issue with the tag `enhancement`
- Describe the feature and why it's useful
- Mention any alternatives you've considered

### Submitting Code
1. **Fork** the repository
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes and test thoroughly
4. Commit with clear messages: `git commit -m "Add: short description"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a **Pull Request**

### Pull Request Checklist
- [ ] Code follows the project's style
- [ ] Self-review of code completed
- [ ] Comments added for complex logic
- [ ] README updated (if needed)
- [ ] No new warnings or errors
- [ ] Tested on your OS

## 🎯 Areas Needing Help

- [ ] **Playlist/Channel support** - Download entire playlists or channels
- [ ] **Download Queue** - Manage multiple downloads
- [ ] **Format Conversion** - Convert to MP3, MP4, AVI, etc.
- [ ] **Speed Optimization** - Improve download performance
- [ ] **UI Improvements** - Better themes, accessibility
- [ ] **Test Coverage** - Add unit tests
- [ ] **Documentation** - More examples, better guides

## 💡 Development Guidelines

### Code Style
- Follow **PEP 8** for Python code
- Use meaningful variable names
- Add docstrings to functions
- Keep functions small and focused

### Testing
- Test on your operating system before submitting
- Check edge cases (invalid URLs, network errors, etc.)
- Ensure GUI remains responsive during downloads

### Commit Messages
- Start with a verb: `Add`, `Fix`, `Update`, `Remove`
- Keep it short but descriptive
- Reference issues if applicable: `Fix #123`

## 🙏 Thank You!

Every contribution, big or small, makes a difference. Whether it's code, documentation, bug reports, or feature suggestions, your help is greatly appreciated.

If you have any questions, feel free to open an issue or reach out.

Happy coding! 🚀
